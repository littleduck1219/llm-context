# LLM Context Manager - 프로젝트 개요

## 문제 정의

LLM 기반 개발(Vibe Coding)을 할 때 발생하는 주요 문제:
- 컴퓨터 종료 또는 세션 종료 시 모든 대화 히스토리 손실
- 새 세션에서 이전 작업 내용을 다시 설명해야 하는 비효율
- 프로젝트별 중요 결정사항, 패턴, 컨벤션 추적 불가
- 긴 히스토리를 LLM 토큰 제한에 맞춰 압축하는 어려움

## 솔루션

LLM 개발 세션의 히스토리와 컨텍스트를 체계적으로 관리하는 도구 제공:
1. **세션 저장/복원**: 대화 히스토리를 영구 저장하고 언제든 이어서 작업
2. **프로젝트 메모리**: 기술 스택, 아키텍처 결정, 코드 패턴 저장
3. **스마트 요약**: 긴 히스토리를 자동 요약하여 토큰 최적화
4. **컨텍스트 내보내기**: 새 LLM 세션에 입력할 수 있는 형식으로 내보내기

## 핵심 기능

### 1. 프로젝트 관리
- 프로젝트 생성/수정/삭제
- 프로젝트별 기술 스택, 설명, 경로 관리
- 프로젝트 메모리 (아키텍처 노트, 펜딩 태스크)

### 2. 세션 관리
- 세션 생성 및 프로젝트 연결
- 대화 메시지 저장 (user/assistant/system)
- 세션별 요약 생성

### 3. 메모리 시스템
- **결정사항(Decisions)**: 중요한 기술적 결정과 이유
- **코드 패턴(Patterns)**: 자주 사용하는 코드 패턴
- **컨벤션(Conventions)**: 프로젝트 코딩 규칙

### 4. 검색
- 모든 메시지에서 키워드 검색
- 프로젝트/세션 필터링

### 5. 내보내기
- Markdown 형식 컨텍스트 생성
- JSON 형식 데이터 백업
- 토큰 제한에 맞춘 자동 압축

## 기술 스택

| 영역 | 기술 |
|------|------|
| Frontend | Next.js 14, React 18, TypeScript |
| Styling | Tailwind CSS 4.x |
| Database | SQLite (better-sqlite3) |
| CLI | Commander.js |
| Icons | Lucide React |
| Date | date-fns |

## 사용자 인터페이스

1. **웹 대시보드** (`npm run dev`)
   - http://localhost:3000
   - 프로젝트/세션 시각화
   - 직관적인 UI로 모든 기능 접근

2. **CLI 도구** (`npm run cli`)
   - 터미널에서 빠른 작업
   - 컨텍스트 생성 및 내보내기
   - 스크립트 자동화 지원

## 데이터 저장 위치

```
~/.llm-context-manager/context.db
```

모든 데이터는 로컬 SQLite 데이터베이스에 저장되어 오프라인에서도 안전하게 사용 가능합니다.
